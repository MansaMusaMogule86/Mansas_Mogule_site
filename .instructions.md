**First time setup:**

**Run locally:**

## Core Architecture Rules

### 🔴 CRITICAL: SSR Only — No Client-Side React
- **All HTML is server-rendered as strings** in `pages.ts`
- Routes are defined in `index.tsx` as Hono handlers
- Pages return complete HTML via `layout()` wrapper function
- **No interactive client JavaScript** — this is a static HTML generator

### 🎨 Design System is Law
- **Never hardcode colors.** Use CSS custom properties from [layout.ts](src/layout.ts):
  - `--color-gold`, `--color-indigo`, `--color-violet`, `--color-blue`, `--color-emerald`
  - `--blur-glass`, `--easing-smooth`, `--duration-normal`, `--shadow-md`, etc.
- Change design tokens **only** in `layout.ts` `:root` section
- All fonts are imported from Google Fonts in `layout.ts`

### 📐 SVG Icon Pattern
All icons follow this structure:
```typescript
const miniIcon = (accent: string, path: string): string => `
  <svg viewBox="0 0 64 64" width="42" height="42" fill="none" aria-hidden="true">
    <rect x="9" y="9" width="46" height="46" rx="14" fill="${accent}14" stroke="${accent}" stroke-opacity=".48"/>
    <rect x="16" y="16" width="32" height="32" rx="9" fill="rgba(255,255,255,.12)" stroke="rgba(255,255,255,.22)"/>
    <path d="${path}" stroke="${accent}" stroke-width="3.1" stroke-linecap="round" stroke-linejoin="round"/>
    <path d="${path}" stroke="#F0D597" stroke-width="1" stroke-linecap="round" stroke-linejoin="round" opacity=".7"/>
  </svg>`
```
- **Must include** `aria-hidden="true"` for decorative icons
- **Always use stroke**, not fill, for line icons
- **Double-stroke effect**: base color + `#F0D597` highlight

## File-by-File Guidelines

### `index.tsx` — Route Definitions
- **One responsibility:** Define Hono routes
- **Pattern:** `app.get('/path', (c) => c.html(getPage('slug', ...args)))`
- **No HTML** should be in this file; call `getPage()` instead
- **Status codes:** Use appropriate HTTP status (default 200)

### `pages.ts` — Page Content + Division Data
- **Export `getPage(slug, ...args)`** factory function returning complete HTML string
- **DIVISIONS array** holds all 6 divisions with structure:
  ```typescript
  { id, n, title, tag, color, color2, glow, emoji, status, bc, desc, services, rev, clients, arr }
  ```
- **Dynamic content:** If you need to render division or page variants, add parameters to `getPage()`
- **Always wrap output:** `return layout(content, { title: '...', description: '...' })`

### `layout.ts` — Design System + Base HTML
- **CSS Custom Properties:** All in `:root` selector (lines 1–50)
- **Global CSS:** Fonts, base styles, custom cursor, animations
- **layout() function:** Returns complete HTML document with inline CSS + content
- **Never export raw CSS** — always wrap in `globalCSS` template string

### `home.ts` — Homepage-Specific Content
- **SVG functions:** Crown, icons, visual assets for homepage only
- **Copy & messaging:** Homepage hero text, section descriptions
- **Exported function:** `homePage` string or `getHomePage()` function
- **Import pattern:** `import { homePage } from './home'`

## Development Workflow

### Adding a New Page

1. **Create function in `pages.ts`:**
   ```typescript
   export function getPage(slug: string, param?: string): string {
     if (slug === 'new-page') {
       const content = `<section>New page content</section>`
       return layout(content, { title: 'New Page', description: 'Page desc' })
     }
   }
   ```

2. **Add route in `index.tsx`:**
   ```typescript
   app.get('/new-page', (c) => c.html(getPage('new-page')))
   ```

3. **Test:**
   - `npm run dev` — Vite should auto-reload
   - `pm2 start ecosystem.config.cjs` — Test full routing

4. **Build and verify:**
   ```bash
   npm run build
   npm run preview  # Visit http://localhost:8787/new-page
   ```

### Adding a New Division

1. **Add entry to `DIVISIONS` array in `pages.ts`:**
   ```typescript
   {
     id: 'moguls-new',
     n: '04',
     title: 'Moguls New',
     tag: 'Your tagline',
     color: '#YOURCOLOR',
     color2: '#LIGHTERCOLOR',
     glow: 'rgba(R,G,B,.28)',
     emoji: miniIcon('#LIGHTCOLOR', 'M...SVG path...'),
     status: 'ACTIVE',
     bc: 'badge-green',
     desc: 'Division description...',
     services: ['Service 1', 'Service 2'],
     rev: 'Revenue focus',
     clients: 'TARGET',
     arr: 'FLAGSHIP'
   }
   ```

2. **Create division detail page in `pages.ts`:**
   ```typescript
   if (slug === 'division' && param === 'moguls-new') {
     const content = `<section>Division detail</section>`
     return layout(content, { title: 'Moguls New', description: '...' })
   }
   ```

3. **Route already exists:** `/divisions/:slug` in `index.tsx` handles all dynamic division pages

### Updating Colors/Design Tokens

1. **Edit `layout.ts` `:root` section:**
   ```css
   :root {
     --color-gold: #NEWCOLOR;
     --color-gold-light: #NEWLIGHTCOLOR;
     /* etc */
   }
   ```

2. **Build and test:**
   ```bash
   npm run build
   npm run preview
   ```

3. **If colors are hardcoded in SVG functions,** search for color hex values in `pages.ts` and `home.ts` and update them

## Debugging Checklist

| Problem | Solution |
|---------|----------|
| Vite dev server blank | Run `npm run build` to check for errors; restart Vite |
| PM2 server won't start | Verify `ecosystem.config.cjs` is valid; run `node dist/index.js` directly |
| CSS not applying | Check `layout.ts` for typos in CSS custom property names; use DevTools Inspector |
| Route 404 on preview | Verify route in `index.tsx` matches requested path |
| TypeScript error | Run `npx tsc --noEmit` to see full error details |
| HTML not rendering | Check that `getPage()` returns wrapped layout; inspect `dist/` HTML structure |
